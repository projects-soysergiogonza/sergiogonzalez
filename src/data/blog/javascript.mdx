**`JavaScript`** es un lenguaje de programación usado en varios entornos, pasando por el Desarrollo Web que es su
enfoque principal, hasta el Desarrollo de Aplicaciones Móviles, Desarrollo de Software, Desarrollo de Bases de Datos,
Desarrollo de VideoJuegos, incluso Desarrollo de tecnologías IoT.

El lenguaje por sí solo trabaja en el lado del cliente, es decir que se ejecuta en el navegador del usuario. Sin
embargo, con el tiempo se han desarrollado tecnologías que permiten ejecutar JavaScript en el lado del servidor,
como es el caso de `Node.js` y `Deno`, donde son motores que permiten ejecutar código `JavaScript` en el
Backend.

####  <span style={{color: "#E44D26"}}>Origen y Evolución</span>
Creado por Brendan Eich en 1995 mientras trabajaba en `Netscape Communications Corporation`. JavaScript fue
originalmente desarrollado bajo el nombre `"Mocha"`, posteriormente renombrado a `"LiveScript"` y finalmente a
`"JavaScript"`. La decisión del nombre final fue principalmente una estrategia de marketing, debido a la popularidad
de `Java` en ese momento.

#### <span style={{color: "#E44D26"}}>Estandarización con ECMAScript</span>
En 1997, `JavaScript` fue estandarizado por `Ecma International` dando lugar a la especificación `ECMAScript`. La
estandarización ha permitido que el lenguaje evolucione de manera más ordenada y predecible, experimentado
varias actualizaciones significativas. Si quieres saber más sobre `ECMAScript`, te invito a leer mi artículo <span
style={{color: "blue", fontWeight: "bold"}}> [¿Qué es ECMAScript?](concat)</span>

#### <span style={{color: "#E44D26"}}>Paradigmas de Programación</span>
`JavaScript` es un lenguaje de programación multiparadigma, lo que significa que soporta varios estilos de
programación como `Programación Orientada a Objetos`, `Programación Funcional` y `Programación Imperativa`. Esto
permite a los desarrolladores elegir el estilo de programación que mejor se adapte a sus necesidades.

#### <span style={{color: "#E44D26"}}>Características Fundamentales</span>
- **`Lenguaje Interpretado`**: Significa que el código se ejecuta línea por línea en tiempo real.
- **`Tipado Dinámico`**: Significa que las variables no tienen un tipo de dato fijo y pueden cambiar
durante la ejecución del programa.
- **`Asincronía`**: Significa que puede ejecutar múltiples tareas al mismo tiempo sin bloquear el hilo
principal, y maneja las operaciones asíncronas mediante callbacks, promesas y async/await
- **`Event-Driven`**: Orientado a Eventos, lo que significa que puede responder a eventos
  como clics de ratón, pulsaciones de teclas, etc.
- **`Prototipado`**: Significa que los objetos heredan propiedades y métodos de otros objetos.
- **`Funciones de Orden Superior`**: Significa que las funciones pueden ser asignadas a variables, pasadas como
argumentos y devueltas como valores.

#### <span style={{color: "#E44D26"}}>Ecosistema y Herramientas</span>
`JavaScript` es uno de los lenguajes más ricos en el mundo del desarrollo de software, con una gran cantidad de
paquetes, librerías y frameworks que facilitan el desarrollo de aplicaciones. Algunas de las herramientas más populares son:


#### <span style={{color: "#E44D26"}}>Gestores de Paquetes</span>
- **`npm`**: Gestor de paquetes por defecto de `Node.js`, es el más popular en el mundo de JavaScript.
- **`yarn`**: Gestor de paquetes alternativo a `npm`, desarrollado por `Facebook`.
- **`pnpm(Mi Favorito)`**: Gestor de paquetes que instala las dependencias en un único lugar, reduciendo el espacio en
disco.

#### <span style={{color: "#E44D26"}}>JavaScript en el Frontend</span>
> Un Framework Frontend es una colección de herramientas, librerías y estructuras predefinidas que facilitan el desarrollo de interfaces de usuario (UI) en aplicaciones web. Proporcionan una base estructurada y componentes reutilizables para construir aplicaciones complejas, haciendo que el proceso de desarrollo sea más eficiente, organizado y escalable.


- **`React`**: Librería desarrollada por `Meta` para la creación de interfaces de usuario.
- **`Next.js`**: Framework de `React` que facilita la creación de aplicaciones web.
- **`Vue`**: Framework progresivo para la creación de interfaces de usuario.
- **`Angular`**: Framework desarrollado por `Google` para la creación de aplicaciones web.
- **`Svelte`**: Framework que compila a `JavaScript` puro, eliminando la necesidad de `Virtual DOM`.
- **`Astro`**: Framework que compila a `JavaScript` puro, eliminando la necesidad de `Virtual DOM`.

#### <span style={{color: "#E44D26"}}>JavaScript en el Backend</span>
Tradicionalmente, JavaScript se asociaba principalmente con el frontend, es decir, con la parte de una aplicación
web que interactúa directamente con el usuario y se ejecuta en el navegador. Sin embargo, gracias a tecnologías como
`Node.js` y `Deno`, ahora es posible ejecutar JavaScript en el servidor, gestionando tareas como:

`Manejo de solicitudes HTTP:` Recibir peticiones desde el cliente y enviar respuestas.\
`Acceso a bases de datos:` Interactuar con bases de datos para almacenar y recuperar información.\
`Procesamiento de datos:` Realizar cálculos, transformaciones y manipulación de datos.\
`Gestión de archivos:` Leer y escribir archivos en el servidor.\
`Creación de APIs:` Exponer funcionalidades del servidor a otros sistemas.

#### <span style={{color: "#E44D26"}}>¿Por qué usar JavaScript en el backend?</span>
- `Un solo lenguaje:` Permite a los desarrolladores utilizar el mismo lenguaje tanto en el frontend como en el
backend, lo que facilita la transición entre ambas partes de la aplicación y reduce la curva de aprendizaje.
- `Ecosistema amplio:` JavaScript cuenta con una gran comunidad y una amplia variedad de librerías y frameworks tanto
para el frontend como para el backend, lo que agiliza el desarrollo.
- `Alto rendimiento:` Node.js, el entorno de ejecución más utilizado para JavaScript en el backend, es conocido por su
excelente rendimiento y escalabilidad, gracias a su modelo de concurrencia basado en eventos.
- `Desarrollo rápido:` La sintaxis sencilla y la gran cantidad de herramientas disponibles hacen que el desarrollo con
JavaScript sea rápido y productivo.
